console.log("Hello,AST!"); 再看右边，网站自动给我们解析好了


可以看到，最顶部是一个根节点，类型是 File，下面的节点解释如下:

1.type: 表示当前节点的类型，我们常用的类型判断方法 t.is****(node),就是判断当前的节点是否为某个类型。**
2.start: 表示当前节点的起始位。
3.end: 表示当前节点的末尾。
4.loc : 表示当前节点所在的行列位置，里面也有 start 与 end 节点，这里的 start 与上面的 start 是不同的，这里的 start 是表示节点所在起始的行列位。置，而 end 表示的是节点所在末尾的行列位置。
5.errors:是 File 节点所特有的属性，可以不用理会。
6.program:包含整个源代码，不包含注释节点。
7.comments:源代码中所有的注释会在这里显示。
在学习的时候，可以重点关注** type,start,end 以及它的子节点**，其他的属性我们在还原的时候一般用不到。


它表示的是对这行代码的解析:
console.log("Hello,AST!");
它是一个** ExpressionStatement 类型的节点，可以看成是一个表达式块，大家多多解析就明白了，这个节点一定会有一个 expression **的子节点，所以我们在构造节点的时候，只需构造它必有的节点即可。至于 start，end 以及 loc，babel 会帮你自动处理好。

这里** expression 节点的类型是 CallExpression，它同时又包含了 callee 和 arguments 两个子节点，callee** 可以理解为函数名，而** arguments **则是它的参数。

callee 节点 包含了 object，property 和 computed 节点，object 表示对象，property 表示其属性，而 computed 表示其方式。

arguments 节点，是一个 Array 类型，因此单词是复数形式。在这里它只有一个元素，说明该函数只有一个实参，实参是一个 StringLiteral 类型的节点，也就是字面量。

讲到这里，相信大家对 AST 有个大概的认识。源代码可以看成是由一个一个的节点组成的，然后 AST 解析源代码就是解析成了一个个的节点，你要做的就是对这些节点进行操作。

也许你会好奇，为什么要给它计算出具体的值。目的很简单，就是方便观看代码。
也方便处理 VariableDeclarator 节点

对于这种表达式，babel 库给了我们一个很强大的方法:path.evaluate ，通过该方法可以直接帮你把结果计算出来，非常的好用。

它的代码在 node_modules@babel\traverse\lib\path\evaluation.js 这个文件里。我们来看源代码:

function evaluate() {
const state = {
confident: true,
deoptPath: null,
seen: new Map()
};
let value = evaluateCached(this, state);
if (!state.confident) value = undefined;
return {
confident: state.confident,
deopt: state.deoptPath,
value: value
};
}

它的核心处理是 evaluateCached 函数，这里我们只需要关心返回值即可。

它返回了一个 object 对象，从代码不看看出，当 confident 为 true 时，value 就是帮你计算出来的值。

因此，插件很好写，假如需要同时遍历 Identifier 和 BinaryExpression 节点:

const visitor = {
“Identifier|BinaryExpression”(path)
{
const {confident,value} = path.evaluate();
confident && path.replaceInline(t.valueToNode(value));
},
}

这样就可以了，代码非常的短，但是功能却是非常的强大。

也许你不知道它到底处理了哪些节点，你可以关注 _evaluate 方法，代码很简单，里面的有节点都可以遍历。

**binding 相关: **
let binding = scope.getBinding(name);
目前我看到的只有 变量 定义 和 函数定义定义拥有 binding，其他的获取 binding 都是 undefined。
例如:
var a = 123; 这里的 a 就拥有 binding。
而
function test(a,b,c) {};
函数名 test 以及形参 a，b，c 均拥有 binding。
bind.path 用于定位最开始 binding 的 path;
binding.constant 用于判断当前变量是否被改变，true 表示不变，false 表示有更改变量值。
binding.referenced 用于判断当前变量是否被引用，true 表示代码下面有引用该变量的地方，false 表示没有地方引用该变量。
注意，引用和改变不共用。
binding.referencePaths 它是一个 Array 类型，包含所有引用的 path，多用于替换。
binding.constantViolations 它是一个 Array 类型，包含所有改变的 path，多用于判断。

var a = 3;

var a = 1 + 2;

再就是注意判断条件，否则很容易无限循环，比如代码中如果不判断 id.name==”a” ，运行是会一直进行插入操作，因为代码插入了一个新的节点后，又遍历到了该节点，满足条件，又可以继续插入，因此会一直这样循环，导致的结果就是内存溢出。

总结:

1.明确目的，需要遍历什么类型的节点以及往该节点进行插入操作 2.构造节点时千万要注意，插入后，节点之间到底是** , 还是 ; ** 3.判断好条件，不要无限循环了。 4.直接插入。
